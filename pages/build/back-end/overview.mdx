Once you run `vastly generate service`, the `services` directory will be populated with your new microservice, a few tools provided for your backend architecture, and some for the [Front End](/build/front-end/overview) in order to connect the two. While we provide tools that are somewhat opinionated, you are more than welcome to customize your backend architecture.

## Breaking down the backend
The Wave generated backend consists of three main elements:
2. A versatile RDS configuration that supports PostgreSQL, MySQL, SQLite, MongoDB, CockRoachDB, and Microsoft SQL Server
1. A ready-to-use GraphQL API template
3. An easy deployment method to AWS AppSync.

### Folder Structure
```bash filename="services/"
project/
  |__ handler.ts  # lambda function handler (API resolver)
  |__ server.ts   # local server (for dev)
  |__ cdk/        # AWS CDK deploy boilerplate
  |__ prisma/
    |__ schema.prisma  # prisma schema (data source)
    |__ generated/     # auto-generated after each `npx prisma generate`
  |__ README.md
```

### Tools
By default, you'll be provided with `prisma` and it's extension, `prisma-appsync`. Prisma.io is a type-safe database toolkit that simplifies access and management, while Prisma AppSync is a plugin that integrates Prisma with AWS AppSync.
